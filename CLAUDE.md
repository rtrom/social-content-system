# Social Content System

A Claude-based system for evaluating content niches and producing coherent, multi-channel social content from a single pillar post. Built as portable Skills, wrapped by Claude Code subagents and slash commands for the technical path, and usable directly via Skills in Cowork for the non-technical path.

## The two tracks

**Track A — Niche Evaluation** ("should we create content in this niche?"). Runs occasionally. Scores candidate niches against a fixed six-criterion rubric and returns a Niche Evaluation Scorecard. Entry point: `/evaluate-niche [niche]`.

**Track B — Content Generation** ("we're in the niche, now produce"). Runs on a recurring cadence. Turns trend signal into one pillar blog post plus derivatives for TikTok, Reels, Shorts, X, Threads, and a newsletter — all filed as one release. Entry points: `/trend-scan` → `/content-plan` → `/full-pipeline`.

## Core principle — pillar-and-repurpose

Every content run produces ONE pillar (the blog post) first, and every other channel is a *derivative* of it — pulled quotes, restructured hooks, compressed points — never an independent take. Eight writers researching separately produce eight contradicting stories; one pillar with eight derivations produces one story eight ways. The `content-brief` is the coherence anchor every channel binds to; `editor-qa` is the final check that nothing contradicts across channels.

## The dependency spine

Nothing downstream is coherent without the two foundation artifacts:
1. **`brand-voice-profile.md`** — generated by running the `brand-voice` skill. Every content skill and subagent reads it. **Generate this first.** The pipeline will stop and flag if it's missing.
2. **The content brief** — generated per run by `content-strategist` / `/content-plan`. Every channel derives from it.

## Layout

- `.claude/skills/` — the 17 portable skills (the actual workflow logic; source of truth)
- `.claude/agents/` — 14 subagents wrapping the skills into the cascade (Claude Code only)
- `.claude/commands/` — 4 slash commands launching the tracks
- `calendar/` — filed releases, one directory per release (see the `content-calendar` skill)

## Conventions

- **Release naming:** `YYYY-MM-DD_<release-slug>_<channel>.md`, where the slug comes from the pillar's URL slug (set by `blog-outline`). Every derivative in a release shares the date and slug so they group together.
- **Model tiering:** opus for synthesis/creative agents (`niche-scout`, `content-strategist`, `blog-writer`), sonnet for writers and research analysts, haiku for `editor-qa`.
- **Cascade rule:** non-leaf subagents carry `Agent` in their tools to spawn children; leaf agents omit it so nesting stops deliberately.

## MCP tools (optional, third-party — not verified by Anthropic)

- **Virlo** (`short-form-data`) — raw trend/hashtag/creator data, feeds `trend-research` and the Track A analysts.
- **Niche** (`sailquery/niche`) — end-to-end editorial pipeline; overlaps with the drafting skills, can run alongside for more control.

Both act within whatever account is authenticated; create accounts yourself and check pricing before wiring into a daily workflow. See `mcp.json.example` to configure.

## Requirements

- Claude Code v2.1.172+ for nested subagents (the cascade). `npm i -g @anthropic-ai/claude-code` or `claude install` if unsure.
- Nested delegation is still experimental — each orchestrating agent names its delegation targets explicitly in its own prompt.
