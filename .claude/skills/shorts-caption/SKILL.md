# Shorts Caption

For YouTube Shorts, the publish text is not a social caption — it is **SEO metadata**: a search-optimized title, a description, and tags. Treat it as YouTube search copy, closer to the pillar's meta title/description than to a TikTok caption. This is the difference that matters: YouTube surfaces Shorts through search and suggested, so this text is a ranking input, not just a frame.

Derive from the same pillar and `content-brief` anchor as the script (apply `repurposing`); write against `brand-voice-profile.md`. Conventions — title length, description structure, tag strategy — are in [`shorts-metadata-reference.md`](shorts-metadata-reference.md).

## Procedure

1. **Write the SEO title.** Keyword-bearing, readable, front-loaded with the search term — like a headline, not a caption. Carry the topic a viewer would search. Completion: title present, within length, primary keyword front-loaded.

2. **Write the description.** Opens with a search-relevant sentence carrying the keyword, then a short expansion of the message, then the brief's CTA and any links. Completion: description opens on the keyword, carries the CTA, structured per the reference.

3. **Set tags.** Keyword tags reflecting the topic and niche, sourced against the YouTube trend signal. Completion: tag set present, on-topic, count within convention.

Note: because these three share DNA with the pillar's own meta title/description/slug (`blog-outline`), keep them consistent with the pillar's SEO framing — same topic, same primary keyword — rather than inventing a divergent one.
