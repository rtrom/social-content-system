# X Copy

Writes the X **derivative** of the pillar — a single post or a thread, within X's character limit. X rewards a sharp standalone line and a thread that earns each next post; the constraint *is* the form.

Apply `repurposing` against the pillar and `content-brief` anchor — X leans on **pull** (a sharp verbatim line from the pillar) and **compress**. Write against `brand-voice-profile.md`. Conventions — character limit, hook post, thread structure — are in [`x-copy-reference.md`](x-copy-reference.md).

## Procedure

1. **Decide single vs. thread.** One sharp point → single post. A key point that needs 3–5 beats → thread. Completion: format chosen to fit the content, not by default.

2. **Write the hook post.** The first post must stand alone and earn the click/expand — pull the pillar's sharpest line or lead with the payoff. Completion: first post works as a standalone and stays within the character limit.

3. **Build the thread (if threading).** Each post carries one beat, each within the limit, each readable alone but building. Land the brief's CTA in the final post. Completion: every post within the limit; the last post carries the CTA; no post is filler.

4. **Check against the pillar.** Every claim traces to the pillar or brief. Completion: no invented claim.
