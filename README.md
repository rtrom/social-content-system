# Social Content System

A Claude-based system for **social media content**. It does two jobs:

- **Track A — decide** whether a content niche is worth pursuing (a scored niche evaluation).
- **Track B — produce** a full multi-channel content set from a single blog post — TikTok, Reels, Shorts, X, Threads, and a newsletter, all derived from one pillar so they tell *one* consistent story.

It works two ways, and you only need the one that fits you: a **point-and-click path** in Claude.ai (for non-developers) and a **technical path** in Claude Code (subagents + slash commands).

## What's inside

- **17 skills** ([`.claude/skills/`](.claude/skills/)) — the actual workflow logic. These are the brains; everything else just wraps them. They work in both Claude.ai and Claude Code.

## Core idea — pillar-and-repurpose

Every Track B run produces **one pillar** (the blog post) first; every other channel is a *derivative* of it — pulled quotes, restructured hooks, compressed points — never an independent take. Eight writers researching separately produce eight contradicting stories; one pillar with eight derivations produces one story eight ways.

Two **foundation artifacts** hold it all together, and both are saved as files that any skill loads on demand:

1. **Brand voice profile** — how everything sounds. Generated by the `brand-voice` skill; every content skill loads it before writing. **Set this up first** — just tell Claude *"Help me set up my brand voice."*
2. **Content brief** — what a given release is about (core message, key points, CTA). Generated per run; every channel writer binds to it.

If either is missing, the system stops and asks rather than guessing.

## Install (Claude Code)

Clone (public repo — HTTPS, no key needed):

```bash
git clone https://github.com/rtrom/social-content-system.git
cd social-content-system
```

**Option A — use the repo as your project.** Open this folder in Claude Code. Everything under `.claude/` is already wired up; nothing to copy.

**Option B — install into another project or globally.** Run the install script:

```bash
# macOS / Linux / Git Bash
./install.sh                 # global: copies into ~/.claude (available in every project)
./install.sh /path/to/proj   # into a specific project's .claude/ (also drops CLAUDE.md)
```
```powershell
# Windows PowerShell
./install.ps1                      # global: ~/.claude
./install.ps1 -Target C:\path\proj # into a specific project
```

The script copies `skills/`, `agents/`, and `commands/` into the target's `.claude/` and (for project installs) `CLAUDE.md` to its root. **Restart Claude Code afterward** so it loads them. Re-run after any `git pull` to update.

Requirements: Claude Code v2.1.172+ for nested subagents (the cascade).

### Run the workflows

```
# Track A — evaluate a niche
/evaluate-niche stay-at-home-parent-cleaning tiktok,reels

# Track B — produce content (review between each step)
/trend-scan [niche]           → see what's trending, pick a direction
/content-plan [chosen angle]  → review the content brief
/full-pipeline                → produce + QA + file the whole release
```

Or run `/full-pipeline [niche]` cold to chain the whole thing at once.

## How a Track B run flows

```
trend-scout  →  content-strategist  →  blog-writer  ──┬─→ tiktok-scriptwriter
 (signal)         (the brief)          (the pillar)   ├─→ reels-scriptwriter
                                                       ├─→ shorts-scriptwriter
                                                       ├─→ x-writer
                                                       ├─→ threads-writer
                                                       └─→ newsletter-writer
                                                              │
                                                         editor-qa
                                                    (checks all together)
```

The blog post is written first; every social post is derived from it, not researched separately. The final QA pass reads everything together to catch any contradiction before it's filed.

## Notes

- **Optional trend tools (Virlo, Niche) are third-party**, not verified by Anthropic. They act within whatever account you authenticate; create the accounts yourself and check pricing before relying on them. See [mcp.json.example](mcp.json.example).
- **Brand voice is the foundation.** If content ever starts sounding off, re-run `brand-voice` — every skill picks up the change automatically.
- **Persistence on Claude.ai:** the voice profile and brief are files. To keep them across separate chats, work inside a Claude **Project** and paste the profile in (the `brand-voice` skill walks you through this).
